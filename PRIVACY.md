# Privacy Policy for Awesome Full View

**Effective date: June 18, 2026**

Awesome Full View is a Chrome extension that lets users retrieve spending transactions from their authenticated Fidelity Full View session and review those transactions in a local spending dashboard.

## Information the extension processes

When the user requests a refresh, the extension processes financial transaction information returned by Fidelity, including transaction dates, descriptions, amounts, account names, categories, subcategories, transaction types, and transaction identifiers.

The extension may also process the date range selected by the user and basic synchronization status or error information. If the user chooses to import a CSV file, the extension processes the transaction information in that file locally.

## How information is used

This information is used only to provide the extension's single purpose: displaying spending totals, trends, categories, recurring expenses, and transaction history, and allowing the user to export filtered transaction data.

The information is not used for advertising, profiling, determining creditworthiness, lending, or any unrelated purpose.

Awesome Full View's use and transfer of user data complies with the Chrome Web Store User Data Policy, including its Limited Use requirements.

## Authentication information

The extension does not collect or store Fidelity usernames, passwords, MFA codes, browser cookies, or session tokens. Authentication takes place directly on Fidelity's website. Requests to Fidelity use the authenticated session already present in the user's browser.

## Storage and transmission

Transaction data and synchronization information are stored locally on the user's device in the extension's browser storage.

The extension communicates with `digital.fidelity.com` only when needed to retrieve the data requested by the user. The developer does not operate a server that receives this data.

No transaction data is shared with the developer or any third party. Fidelity is the source of the data and receives only the requests necessary to provide its service. The extension does not sell user data, share it with advertisers or data brokers, or use analytics or tracking services.

## Data retention and deletion

Locally stored transaction data remains on the user's device until it is replaced by a later refresh or the extension's local data is cleared.

Users can delete the stored data by removing the extension or by clearing the extension's site data in Chrome.

## Permissions

The extension's access is limited to the permissions needed to find or open the Fidelity Full View Spending page, communicate with Fidelity's spending service, and open the local dashboard. It does not access unrelated website content or use browsing activity for tracking.

## Children's privacy

Awesome Full View is not directed to children under 13, and the developer does not knowingly collect personal information from children.

## Changes to this policy

This policy may be updated if the extension's data practices change. Any updated version will be published on this page with a revised effective date.

## Contact

For privacy questions or requests, open an issue in the [Awesome Full View GitHub repository](https://github.com/justoy/awesome-fullview/issues).
